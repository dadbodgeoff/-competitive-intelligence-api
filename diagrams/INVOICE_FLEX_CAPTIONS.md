```
Just finished building an invoice processing pipeline and wanted to share the real performance numbers.

**What it does:**
- Upload PDF invoice
- AI extracts line items, prices, quantities
- Fuzzy matches to existing inventory
- Saves to database with price tracking
- Real-time progress updates via SSE

**Actual metrics from production:**
- AI parsing: 64 seconds (Gemini 2.5 Flash)
- Items extracted: 15 line items
- Database save: 723ms (includes fuzzy matching)
- Cost: $0.0006 per invoice
- Tokens used: 3,534

**Tech stack:**
- Backend: FastAPI (Python) with async
- Frontend: React + TypeScript
- AI: Gemini 2.5 Flash
- Database: PostgreSQL
- Auth: HTTPOnly cookies
- Real-time: Server-Sent Events (SSE)

**Cool features:**
1. Fuzzy matching uses Levenshtein distance to automatically link invoice items to inventory
2. SSE streaming shows progress in real-time (not polling!)
3. HTTPOnly cookies for XSS protection
4. Malware scanning on upload
5. Transaction-based saves (all or nothing)

**Challenges:**
- Spent 8 hours debugging HTTPOnly cookie auth (worth it for security)
- Fuzzy matching needed tuning to avoid false positives
- SSE streaming required careful error handling

Diagram attached shows the flow with real timing data.

Happy to answer questions!
```
